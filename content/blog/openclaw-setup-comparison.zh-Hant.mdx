## 傳統 OpenClaw 安裝:完整流程

### 開始前的準備

1. **VPS 伺服器** - 從 AWS、DigitalOcean 或 Hetzner 購買並配置
   - 最低要求:2 vCPU、4GB RAM
   - 成本:$5-10/月(例如:Hetzner cx23)
   - **時間:15-30分鐘**(帳戶設定、付款、伺服器配置)

2. **技術前提條件**
   - SSH 客戶端配置
   - 基本的 Linux 命令列知識
   - 了解 systemd 服務
   - 熟悉環境變數

### 分步安裝時間線

#### 階段1:伺服器準備(30-60分鐘)
```bash
# SSH 連接到伺服器
ssh root@your-server-ip

# 更新系統套件
apt update && apt upgrade -y

# 安裝 Node.js 22+(必需 - 舊版本無法工作)
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt install -y nodejs

# 安裝 pnpm
npm install -g pnpm

# 安裝 Docker(可選 - 僅在需要沙盒模式時)
curl -fsSL https://get.docker.com | sh
```

**潛在問題:**
- 需要手動解決的套件衝突
- 防火牆配置阻止安裝
- Node.js 版本錯誤(必須是22+)

#### 階段2:OpenClaw 安裝(20-40分鐘)
```bash
# 全域安裝 OpenClaw CLI
npm install -g openclaw@latest

# 初始化配置
openclaw onboard --non-interactive --mode local
```

**必須配置的項目:**
- Telegram 機器人令牌
- 使用者 ID 白名單(格式:`tg:123456`)
- AI API 金鑰(OpenAI、Z.ai 或自訂)
- 閘道認證令牌
- 網路繫結設定
- 頻道配置

#### 階段3:安全加固(60-90分鐘)

這是**大多數使用者失敗的地方**。根據最近的安全稽核,**超過30,000個 OpenClaw 實例因配置錯誤而暴露在網上**。

**關鍵安全步驟:**
1. **存取控制配置**
   ```bash
   openclaw config set --json channels.telegram \
     '{"enabled":true,"dmPolicy":"allowlist","allowFrom":["tg:YOUR_USER_ID"]}'
   ```

2. **網路安全**
   - 繫結到 loopback(`127.0.0.1`)而非`0.0.0.0`
   - 設定 Tailscale 或 VPN 進行遠端存取
   - 配置防火牆規則
   - 啟用強認證令牌

3. **工具沙盒化**(需要 Docker)
   ```json
   {
     "sandbox": {
       "mode": "all",
       "scope": "agent",
       "workspaceAccess": "ro"
     }
   }
   ```

4. **執行安全稽核**
   ```bash
   openclaw security audit --deep --fix
   ```

**常見安全錯誤:**
- 使用`dmPolicy: "open"`(允許任何人控制你的機器人)
- 在`0.0.0.0`上暴露閘道而不進行認證
- 以明文形式儲存 API 金鑰而不設定檔案權限
- 不啟用工具沙盒化
- 跳過安全稽核

#### 階段4:服務設定和監控(30-45分鐘)

```bash
# 建立 systemd 服務
cat > /etc/systemd/system/openclaw.service <<EOF
[Unit]
Description=OpenClaw Gateway
After=network.target

[Service]
Type=simple
User=openclaw
WorkingDirectory=/opt/openclaw
EnvironmentFile=/opt/openclaw/.env
ExecStart=/usr/bin/openclaw gateway --port 18789
Restart=on-failure

[Install]
WantedBy=multi-user.target
EOF

# 啟用並啟動服務
systemctl enable openclaw
systemctl start openclaw

# 監控日誌
journalctl -u openclaw -f
```

**需要持續維護:**
- 監控錯誤日誌
- 處理崩潰和重啟
- 更新相依性
- 修補安全漏洞
- 管理磁碟空間
