很多人尝试在自己的电脑上安装 OpenClaw，但发现想删掉它时却“删不干净”。因为 OpenClaw 不仅仅是一个简单的软件，它运行时还会悄悄在你的电脑里留下很多隐藏的数据（比如它的记忆、运行记录以及各种配置）。

如果你已经决定不再在自己的电脑上运行它，通过下面的简单步骤，就可以把 OpenClaw 留下的痕迹彻底清理干净。

## 步骤 1：停止并基础卸载

首先，我们要告诉 OpenClaw 停止当前的所有工作，并把它本身的服务从电脑上卸载掉。

打开终端（Terminal），输入下面这行命令并回车：

```bash
openclaw uninstall --all --yes
```

**为什么要做这一步？**  
这不仅会卸载它的主程序，还会一并把它正在后台偷偷运行的服务全部关掉。

## 步骤 2：删除命令行工具

刚刚那一步关掉了服务，但我们在终端里敲击 `openclaw` 时，系统可能仍然能识别它。为了彻底移除这个命令，我们需要删掉它的启动程序。

你可以直接输入：

```bash
rm -f $(which openclaw)
```

**为什么要做这一步？**  
因为不同的人安装 OpenClaw 的方式不一样（不管是 npm 还是 pnpm），用这行命令最直接：它会自动找到 OpenClaw 命令藏在系统的哪个角落，然后“精准击杀”。

> **如何确认删干净了？**  
> 再次输入 `openclaw` 并回车，如果终端提示你 `command not found`（找不到命令），就说明卸载成功了。

## 步骤 3：清理隐藏的缓存数据

即便上面两步都做完了，OpenClaw 依然会在你的电脑里留下大量的“生活痕迹”——比如它的记忆库、工作缓存等等这些隐藏文件夹。放着不管很占空间。

为了腾出电脑空间，请把下面这段命令一次性复制、粘贴到终端里，然后按回车：

```bash
rm -rf ~/.openclaw ~/.clawdbot ~/.moltbot ~/.molthub ~/clawd
```

**为什么要做这一步？**  
这行命令会一次性把你电脑里所有可能被 OpenClaw 霸占的隐藏文件夹通通删掉。注意：这里面的历史记录删掉就找不回来了，请确认你不再需要它们。

## 步骤 4：检查并撤销账号授权

最后一点经常被人忽略：如果你之前在 OpenClaw 里绑定过你的 GitHub、Gmail 或 Google Drive 等账号，卸载软件是**不会**帮你自动解绑的！

**你需要做什么？**  
为了账号安全，请手动登录到 GitHub、Google 等平台的设置页面，找到对应的“已授权应用”列表，把关于 OpenClaw 的授权撤销掉。
