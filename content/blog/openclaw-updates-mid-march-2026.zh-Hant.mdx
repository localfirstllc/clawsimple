OpenClaw 在三月的上半個月推進得非常快。從 3 月初的 [`2026.3.8`](https://github.com/openclaw/openclaw/releases/tag/v2026.3.8) 一路發布到中旬的 [`2026.3.13`](https://github.com/openclaw/openclaw/releases/tag/v2026.3.13-1)。

相比零碎的版本日誌，這裡為你整理了從 `3.8` 到 `3.13` 之間，普通用戶最能直接感受到的幾個核心變化。

---

## 終於吃滿了 GPT-5.4 的百萬上下文，Perplexity 搜尋也回歸了

隨著新模型發布，`2026.3.8` 版本在底層做了一次關鍵的相容性調整：
- **完整支援 GPT-5.4**：不再沿用舊版 Codex 的長度限制，徹底放開了高達 1,050,000 token 的上下文視窗，處理超長文件或巨型程式碼庫時不再會無故被截斷。
- **Perplexity 搜尋回歸**：全面恢復了對 Perplexity OpenRouter / Sonar 模型的支援。無論你是填原生 API Key 還是走代理，現在都能直接讓 Agent 連上 Perplexity 進行高品質網頁檢索了。

---

## 聊天介面順了不少，Mac 語音也更穩了

這半個月用戶端體驗有幾個明顯的改善：

- 工具執行很多的時候，聊天記錄不會頻繁整頁重載了。
- 很長的回覆不會再擠成一塊難讀的大灰色區域了。
- `2026.3.12` 重構了 dashboard 的佈局，聊天、配置、會話管理更集中。
- Mac 用戶的福音：`2026.3.8` 修復了 macOS 上 VoiceWake 和 Talk 語音覆蓋層的閃退問題，直接動嘴聊天的體驗變得更好。

這類改動不算炫技，但能解決「介面一頓一頓、訊息看著費勁」的痛點。

---

## Slack 終於能發按鈕了，Teams 權限更嚴密

如果你把 OpenClaw 接到了辦公軟體，這段時間有幾個好消息：

- **Slack**：`2026.3.12` 和 `3.13` 終於加入了 Slack Block Kit 的支援。機器人的回覆不再只是一坨純文字，而是可以帶有按鈕、卡片、下拉選單。在團隊裡用來做確認和審批，它終於像個真正的工作助手了。
- **MS Teams**：`2026.3.8` 修復了白名單機制，現在即使配置了頻道和群組級許可，也不會意外將群裡的陌生人放行，安全性大幅提高。

---

## Telegram 和 Discord 補上了一大批坑

渠道穩定性經過幾個大版本的修補，現在更可靠了：

- **Telegram**：收圖和檔案的下載鏈路繼續補強；改進了重連時的清理邏輯，不再會出現「機器人重啟後，重複收到舊訊息」的問題。
- **Discord**：回覆拆分、狀態回饋、路由穩定性都在持續修。
- `/reset` 之後，原來的帳號和執行緒資訊會盡量保住，不容易「聊著聊著串線」。

如果你之前遇到過圖片處理失敗、長回覆切得亂、群聊重複發訊息等問題，這一波大體都在往這些地方打補丁。

---

## 定時任務更值得重新試一次了

很多人用 OpenClaw 是為了讓它每天自動跑早報、搞彙總。從 `3.8` 開始，定時任務（Cron）連續做了幾輪大修：

- 修復了因為伺服器重啟導致的漏發問題（現在恢復後會合理補發，不再一窩蜂擠爆或直接跳過）。
- `2026.3.11` 處理了重試、狀態記錄邊界。
- 修復了嵌套 cron 導致的系統死鎖。

簡單說：如果你之前覺得定時任務「偶爾會抽風」，這段時間的更新就是在集中修這個。有些以前不太敢託付給它的日常工作，可以重新試一遍了。

---

## 手機端用起來更像個正式 App 了

移動端在最近的版本裡也有明顯迭代：

- iOS 補了更完整的首開引導和首頁佈局，工具列更適配小螢幕。
- Android 重做了聊天設定頁，連線、語音、媒體這些選項整理得更清楚。
- onboarding（初始化配對）時的掃碼和連線流程也更順暢。

這決定了你願不願意把它當成每天都會打開幾次的手機 App，而不是離開了電腦就沒法用的東西。

---

## 自架用戶的小彩蛋

如果你是自己部署的，除了 Docker 現在支援 `OPENCLAW_TZ` 時區設定以外，`3.8` 版本還自動檢測 SELinux 權限並修復了 Podman 掛載失敗（`Permission denied`）的問題。折騰環境的門檻又降低了一點。

---

## ClawSimple 用戶不需要做什麼

新版本我們會在合適的時機幫你自動更新。如果你想馬上用上最新的大視窗模型和這些互動最佳化，到 Profile 頁面 Relaunch 一下就好了。
