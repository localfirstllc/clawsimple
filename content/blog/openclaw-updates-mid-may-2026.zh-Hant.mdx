OpenClaw 最近一批公開發佈在四月初落地——從 [`2026.4.1`](https://github.com/openclaw/openclaw/releases/tag/v2026.4.1) 到 [`2026.4.8`](https://github.com/openclaw/openclaw/releases/tag/v2026.4.8)——內容非常豐富。距離我們[四月初的回顧](/blog/openclaw-updates-early-april-2026)，功能密度只增不減。

以下是具體內容。

---

## QA Lab：完整的互動式測試套件

這一輪最大的新增功能是 QA Lab——一套完整的 Agent 行為測試和評估框架：

- **即時套件執行器**，配**類 Slack 聊天 UI**，支援淺色/深色模式
- **一鍵 Docker 啟動器**，用於隔離的測試環境
- **基於倉庫的場景執行器**，用真實工作流來測試你的 Agent
- **Frontier 模型對比迴圈**，並排比較不同模型的效能
- **附件理解場景**，用於測試多模態 Agent 行為

如果你曾經懷疑自己的 Agent 是否能正確處理邊緣情況，QA Lab 給你一套系統化的驗證方式——而且不會在你的真實聊天頻道裡刷屏。

---

## Memory Wiki：信念層、摘要和健康報告

OpenClaw 的記憶系統進行了一次重要的架構升級：

- **信念層摘要**讓 Agent 能夠編譯和調和它隨時間推移「知道」的內容
- **健康報告**讓你一眼看清記憶品質和覆蓋範圍
- **編譯摘要提示詞**在不膨脹上下文視窗的情況下提高檢索相關性
- **REM 預覽和安全晉升回放**讓審核和提交記憶變更更加安全

實際效果：Agent 越用越有用，而不是逐漸忘記學過的東西。

## 媒體生成管線：影片、音樂和 Comfy 工作流

媒體能力大幅擴充套件：

- **Runway 影片生成**提供商，支援解耦的非同步任務追蹤
- **音樂生成工具**，用於創意和音訊工作流
- **Comfy 工作流支援**，用於自定義影像生成管線
- **Vydra 媒體提供商**，提供更多生成選項
- **顯式媒體提供商能力宣告**，Agent 知道哪個提供商能處理哪種請求

影片和音樂生成任務現在以背景追蹤任務的方式執行——Agent 發出任務，完成後回報，不會阻塞對話。

---

## 六個新模型提供商和更好的模型檢測

模型目錄持續增長：

- **Arcee AI**——新提供商外掛，支援 OpenRouter 和入門指引
- **Gemma 4**——Google 最新模型，跨提供商自動解析，無需手動指定託管方
- **Bedrock IAM 憑證認證**，透過 `@aws/bedrock-token` 用於 AWS 原生環境
- **Bedrock 嵌入提供商**，透過 Amazon 基礎設施進行記憶搜尋
- **Bedrock Guardrails**，用於內容安全策略
- **Ollama 視覺檢測**——自動透過 `/api/show` 檢測視覺能力，本地視覺模型無需手動配置即可使用

---

## 搜尋、推理和 Gateway 改進

工具層面：

- **SearXNG** 加入 web 搜尋提供商陣營，提供可自託管且注重隱私的搜尋後端
- **Infer CLI**——一等公民 `infer` 命令，可直接在終端中執行推理工作流
- **壓縮檢查點**，帶可插拔的提供商登錄檔，讓長時間執行的會話更省記憶體
- **可配置的聊天歷史**——在 Gateway 中設定你自己的最大字元數限制
- **聊天原生任務看板**，用於行內追蹤 Agent 任務
- **提示快取執行時上下文**暴露給上下文引擎，重複推理更快更便宜

---

## Agent 控制和桌面端改進

- **每個 Agent 的提示詞覆蓋和心跳控制**——無需編輯配置檔案即可微調行為
- **全域性預設提供商引數** (`agents.defaults.params`)，讓所有 Agent 繼承一致的模型設定
- **macOS Voice Wake** 現在可以觸發 Talk Mode，免持對話更流暢
- **iOS Gateway** 顯示更清晰的連線錯誤資訊，而非靜默失敗
- **Cron `--tools` 標誌**，支援按任務的工具白名單——比如計劃一個只能用 web 搜尋但不能用 shell 的研究任務
- **GitHub App 輔助工具** (`gh-read`)，用於 Agent 輔助的程式碼審查工作流

---

## ClawSimple 平臺更新

在託管服務方面，過去一個月 [ClawSimple](/) 執行 OpenClaw 的方式有了幾項改進：

- **自動釋出檢查回滾**——如果新 OpenClaw 版本驗證失敗，部署會自動回滾到上一個已知可用版本，而不是一直處於故障狀態
- **部署健康監控**——基於 SSH 的持續健康探針在問題影響到你之前就發現它們
- **磁碟壓力報告**——平臺現在會在磁碟空間嚴重不足時發出警告並自動清理
- **圖表工作流技能**——新的內建流程圖和技術圖表技能，靈感 UI 中提供 SVG 預覽
- **支援收件箱 Cron Gate**——支援 AI 只在實際有新郵件需要處理時才啟用，保持效率

---

一如既往，ClawSimple 管理的 Agent 會自動獲得這些更新。如果你想立刻用上最新版本，前往你的 Profile 頁面點選 **Relaunch**。
