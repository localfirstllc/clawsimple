[ClawSimple](/) で最初の OpenClaw サーバーをデプロイした後、異なる目的のために複数の独立したボットを実行したくなるかもしれません。嬉しいお知らせがあります。**追加のサーバー費用を支払ったり、複雑な設定を行ったりすることなく、同じアクティブなサーバー（Seat）にさらに多くのエージェントを簡単にアタッチできます**。

以下に、ダッシュボードから新しいエージェントを追加する方法を、簡単な画像付きで説明します。

## ステップ 1: Profile コンソールパネルにアクセスする

ClawSimple コンソールにログインし、**Profile > Overview** ページに移動します。ここでは、現在アクティブなすべてのデプロイプラン（Active Plans）を確認できます。

新しいエージェントをホストしたいサーバーカードを見つけ、カードの右上にある **"+ Add Agent"** ボタンをクリックします。

![Add Agent ボタンの場所](/images/blog/add-agent-button.png)

## ステップ 2: 新しいエージェントを設定する

"+ Add Agent" ボタンをクリックすると、設定モーダルが表示されます。ここでは、新しいボットの基本情報を入力するだけです。

この新しいエージェントは既存のサーバーにアタッチされるため、一部の共通設定（アクセス制限など）を引き継ぎますが、以下の情報を提供する必要があります：

1. **Telegram Bot Token（ボットトークン）**：Telegram の `@BotFather` から取得した、新しいボットの API トークンです。
2. **AI Model（AI モデル）**：この特定のボットで使用したい AI モデルをドロップダウンリストから選択します（同じサーバー上の異なるボットで別々のモデルを実行できます！）。
3. **Allowed User ID（許可されたユーザー ID）**（任意/デフォルト）：通常はメインアカウントの ID がデフォルト設定されます。他人のために設定する場合やアクセスを制限したい場合は、その人の Telegram ユーザー ID を入力してください。

![エージェント設定の入力](/images/blog/add-agent-modal.png)

## ステップ 3: 接続されるのを待つ

入力情報が正しいことを確認したら、**"Add agent"** ボタンをクリックして送信します。ClawSimple のバックエンドシステムが、新しい設定を稼働中のサーバーに自動的にプッシュします。

新しく追加されたボットは、すぐに "Active"（アクティブ）とマークされます。そうなれば、Telegram ですぐに新しいアシスタントへのメッセージ送信を開始できます。

当社の効率的な基盤アーキテクチャのおかげで、リクエスト制限内であれば、1 つのサーバーで最大 10 個の独立したボットを快適にホストできます。ぜひ試してみて、次なる特化型 AI アシスタントを今すぐデプロイしましょう！
