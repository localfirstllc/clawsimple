OpenClaw 最近一批公开发布在四月初落地——从 [`2026.4.1`](https://github.com/openclaw/openclaw/releases/tag/v2026.4.1) 到 [`2026.4.8`](https://github.com/openclaw/openclaw/releases/tag/v2026.4.8)——内容非常丰富。距离我们[四月初的回顾](/blog/openclaw-updates-early-april-2026)，功能密度只增不减。

以下是具体内容。

---

## QA Lab：完整的交互式测试套件

这一轮最大的新增功能是 QA Lab——一套完整的 Agent 行为测试和评估框架：

- **实时套件运行器**，配**类 Slack 聊天 UI**，支持浅色/深色模式
- **一键 Docker 启动器**，用于隔离的测试环境
- **基于仓库的场景运行器**，用真实工作流来测试你的 Agent
- **Frontier 模型对比循环**，并排比较不同模型的性能
- **附件理解场景**，用于测试多模态 Agent 行为

如果你曾经怀疑自己的 Agent 是否能正确处理边缘情况，QA Lab 给你一套系统化的验证方式——而且不会在你的真实聊天频道里刷屏。

---

## Memory Wiki：信念层、摘要和健康报告

OpenClaw 的记忆系统进行了一次重要的架构升级：

- **信念层摘要**让 Agent 能够编译和调和它随时间推移"知道"的内容
- **健康报告**让你一眼看清记忆质量和覆盖范围
- **编译摘要提示词**在不膨胀上下文窗口的情况下提高检索相关性
- **REM 预览和安全晋升回放**让审核和提交记忆变更更加安全

实际效果：Agent 越用越有用，而不是逐渐忘记学过的东西。

## 媒体生成管线：视频、音乐和 Comfy 工作流

媒体能力大幅扩展：

- **Runway 视频生成**提供商，支持解耦的异步任务跟踪
- **音乐生成工具**，用于创意和音频工作流
- **Comfy 工作流支持**，用于自定义图像生成管线
- **Vydra 媒体提供商**，提供更多生成选项
- **显式媒体提供商能力声明**，Agent 知道哪个提供商能处理哪种请求

视频和音乐生成任务现在以后台跟踪任务的方式运行——Agent 发出任务，完成后回报，不会阻塞对话。

---

## 六个新模型提供商和更好的模型检测

模型目录持续增长：

- **Arcee AI**——新提供商插件，支持 OpenRouter 和入门指引
- **Gemma 4**——Google 最新模型，跨提供商自动解析，无需手动指定托管方
- **Bedrock IAM 凭证认证**，通过 `@aws/bedrock-token` 用于 AWS 原生环境
- **Bedrock 嵌入提供商**，通过 Amazon 基础设施进行记忆搜索
- **Bedrock Guardrails**，用于内容安全策略
- **Ollama 视觉检测**——自动通过 `/api/show` 检测视觉能力，本地视觉模型无需手动配置即可使用

---

## 搜索、推理和 Gateway 改进

工具层面：

- **SearXNG** 加入 web 搜索提供商阵营，提供可自托管且注重隐私的搜索后端
- **Infer CLI**——一等公民 `infer` 命令，可直接在终端中运行推理工作流
- **压缩检查点**，带可插拔的提供商注册表，让长时间运行的会话更省内存
- **可配置的聊天历史**——在 Gateway 中设置你自己的最大字符数限制
- **聊天原生任务看板**，用于行内跟踪 Agent 任务
- **提示缓存运行时上下文**暴露给上下文引擎，重复推理更快更便宜

---

## Agent 控制和桌面端改进

- **每个 Agent 的提示词覆盖和心跳控制**——无需编辑配置文件即可微调行为
- **全局默认提供商参数** (`agents.defaults.params`)，让所有 Agent 继承一致的模型设置
- **macOS Voice Wake** 现在可以触发 Talk Mode，免提对话更流畅
- **iOS Gateway** 显示更清晰的连接错误信息，而非静默失败
- **Cron `--tools` 标志**，支持按任务的工具白名单——比如计划一个只能用 web 搜索但不能用 shell 的研究任务
- **GitHub App 辅助工具** (`gh-read`)，用于 Agent 辅助的代码审查工作流

## ClawSimple 平台更新

在托管服务方面，过去一个月 [ClawSimple](/) 运行 OpenClaw 的方式有了几项改进：

- **自动发布检查回滚**——如果新 OpenClaw 版本验证失败，部署会自动回滚到上一个已知可用版本，而不是一直处于故障状态
- **部署健康监控**——基于 SSH 的持续健康探针在问题影响到你之前就发现它们
- **磁盘压力报告**——平台现在会在磁盘空间严重不足时发出警告并自动清理
- **图表工作流技能**——新的内置流程图和技术图表技能，灵感 UI 中提供 SVG 预览
- **支持收件箱 Cron Gate**——支持 AI 只在实际有新邮件需要处理时才激活，保持效率

---

一如既往，ClawSimple 管理的 Agent 会自动获得这些更新。如果你想立刻用上最新版本，前往你的 Profile 页面点击 **Relaunch**。
