很多人嘗試在自己的電腦上安裝 OpenClaw，但發現想刪掉它時卻「刪不乾淨」。因為 OpenClaw 不僅僅是一個簡單的軟體，它運行時還會悄悄在你的電腦裡留下很多隱藏的數據（比如它的記憶、運行記錄以及各種設定檔）。

如果你已經決定不再在自己的電腦上運行它，透過下面的簡單步驟，就可以把 OpenClaw 留下的痕跡徹底清理乾淨。

## 步驟 1：停止並基礎卸載

首先，我們要告訴 OpenClaw 停止當前的所有工作，並把它本身的服務從電腦上卸載掉。

打開終端機（Terminal），輸入下面這行命令並按下 Enter：

```bash
openclaw uninstall --all --yes
```

**為什麼要做這一步？**  
這不僅會卸載它的主程式，還會一併把它正在後台偷偷運行的服務全部關掉。

## 步驟 2：刪除命令列工具

剛剛那一步關掉了服務，但我們在終端機裡敲擊 `openclaw` 時，系統可能仍然能識別它。為了徹底移除這個命令，我們需要刪掉它的啟動程式。

你可以直接輸入：

```bash
rm -f $(which openclaw)
```

**為什麼要做這一步？**  
因為不同的人安裝 OpenClaw 的方式不一樣（不管是 npm 還是 pnpm），用這行命令最直接：它會自動找到 OpenClaw 命令藏在系統的哪裡，然後「精準擊殺」。

> **如何確認刪乾淨了？**  
> 再次輸入 `openclaw` 並按下 Enter，如果終端機提示你 `command not found`（找不到命令），就說明卸載成功了。

## 步驟 3：清理隱藏的快取數據

即便上面兩步都做完了，OpenClaw 依然會在你的電腦裡留下大量的「生活痕跡」——比如它的記憶庫、工作快取等等這些隱藏資料夾。放著不管很佔用空間。

為了騰出電腦空間，請把下面這段命令一次性複製、貼上到終端機裡，然後按下 Enter：

```bash
rm -rf ~/.openclaw ~/.clawdbot ~/.moltbot ~/.molthub ~/clawd
```

**為什麼要做這一步？**  
這行命令會一次性把你電腦裡所有可能被 OpenClaw 霸佔的隱藏資料夾通通刪掉。注意：這裡面的歷史記錄刪掉就找不回來了，請確認你不再需要它們。

## 步驟 4：檢查並撤銷帳號授權

最後一點常常被人忽略：如果你之前在 OpenClaw 裡綁定過你的 GitHub、Gmail 或 Google Drive 等帳號，卸載軟體是**不會**幫你自動解除綁定的！

**你需要做什麼？**  
為了帳號安全，請手動登入到 GitHub、Google 等平台的設定頁面，找到對應的「已授權應用程式」列表，把關於 OpenClaw 的授權撤銷掉。
