多くの人が自分のコンピュータにOpenClawをインストールしようと試みますが、削除しようとすると「きれいさっぱり消えない」ことに気づきます。なぜなら、OpenClawは単純なソフトウェアではなく、実行中にバックグラウンドで（メモリ、ログ、設定ファイルなどの）多くの隠しデータをコンピュータに残すからです。

すでにローカル環境での実行をやめることを決めた場合、以下の簡単な手順で、OpenClawが残した痕跡を完全に消去できます。

## ステップ 1: 停止と基本的なアンインストール

まず、OpenClawに現在のすべてのタスクを停止させ、サービスそのものをコンピュータからアンインストールするよう指示します。

ターミナル（Terminal）を開き、以下のコマンドを入力してEnterキーを押します：

```bash
openclaw uninstall --all --yes
```

**なぜこの手順が必要ですか？**  
これにより、メインプログラムがアンインストールされるだけでなく、バックグラウンドで動作しているサービスもすべて強制的に終了されます。

## ステップ 2: コマンドラインツールの削除

前の手順でサービスは停止しましたが、ターミナルで `openclaw` と入力すると、システムがまだそれを認識する場合があります。このコマンドを完全に削除するためには、実行ファイルを削除する必要があります。

以下のように直接入力してください：

```bash
rm -f $(which openclaw)
```

**なぜこの手順が必要ですか？**  
ユーザーによってOpenClawのインストール方法（npmやpnpmなど）が異なるため、このコマンドが最も効果的です。システム内のどこにOpenClawコマンドが隠れているかを自動的に探し出し、ピンポイントで削除します。

> **削除が成功したか確認するには？**  
> 再度 `openclaw` と入力してEnterキーを押します。ターミナルに `command not found`（コマンドが見つかりません）と表示されたら、無事に削除されています。

## ステップ 3: 隠しキャッシュデータの削除

上記の2つの手順を終えた後でも、OpenClawはコンピュータ上にメモリやワークスペースといった大量の隠しフォルダを残しています。これらを放置するとディスクの空き容量を無駄に消費します。

空き容量を確保するために、以下のコマンドをコピーしてターミナルに貼り付け、Enterキーを押してください：

```bash
rm -rf ~/.openclaw ~/.clawdbot ~/.moltbot ~/.molthub ~/clawd
```

**なぜこの手順が必要ですか？**  
これにより、OpenClawが占有している可能性のある隠しフォルダを一括で削除できます。注意：この操作で過去の履歴データは完全に消去され、復元できなくなりますので、本当に不要であることを確認してください。

## ステップ 4: アカウント連携の確認と解除

最後に見落とされがちなポイントです：過去にOpenClawでGitHub、Gmail、あるいはGoogleドライブなどのアカウントを連携していた場合、ソフトウェアをアンインストールしても**自動的に連携は解除されません**！

**どうすればいいですか？**  
アカウントのセキュリティを守るため、GitHubやGoogleなどの設定ページにアクセスし、「連携済みのアプリ」などの一覧からOpenClawのアクセス権限を手動で取り消してください。
