## 従来のOpenClawインストール:完全なプロセス

### 必要なもの(開始前)

1. **VPSサーバー** - AWS、DigitalOcean、Hetznerから購入・設定
   - 最小要件:2 vCPU、4GB RAM
   - コスト:月$5-10(例:Hetzner cx23)
   - **時間:15〜30分**(アカウント設定、支払い、サーバープロビジョニング)

2. **技術的前提条件**
   - SSHクライアント設定
   - 基本的なLinuxコマンドラインの知識
   - systemdサービスの理解
   - 環境変数の知識

### ステップバイステップのインストールタイムライン

#### フェーズ1:サーバー準備(30〜60分)
```bash
# サーバーにSSH接続
ssh root@your-server-ip

# システムパッケージを更新
apt update && apt upgrade -y

# Node.js 22+をインストール(必須 - 古いバージョンは動作しません)
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt install -y nodejs

# pnpmをインストール
npm install -g pnpm

# Dockerをインストール(オプション - サンドボックスモードが必要な場合のみ)
curl -fsSL https://get.docker.com | sh
```

**潜在的な問題:**
- 手動解決が必要なパッケージの競合
- インストールをブロックするファイアウォール設定
- 誤ったNode.jsバージョン(22+が必須)

#### フェーズ2:OpenClawインストール(20〜40分)
```bash
# OpenClaw CLIをグローバルにインストール
npm install -g openclaw@latest

# 設定を初期化
openclaw onboard --non-interactive --mode local
```

**設定が必要な項目:**
- Telegramボットトークン
- ユーザーID許可リスト(形式:`tg:123456`)
- AI APIキー(OpenAI、Z.ai、またはカスタム)
- ゲートウェイ認証トークン
- ネットワークバインディング設定
- チャンネル設定

#### フェーズ3:セキュリティ強化(60〜90分)

ここで**ほとんどのユーザーが失敗します**。最近のセキュリティ監査によると、**30,000以上のOpenClawインスタンスが設定ミスでオンライン公開されています**。

**重要なセキュリティ手順:**
1. **アクセス制御設定**
   ```bash
   openclaw config set --json channels.telegram \
     '{"enabled":true,"dmPolicy":"allowlist","allowFrom":["tg:YOUR_USER_ID"]}'
   ```

2. **ネットワークセキュリティ**
   - `0.0.0.0`ではなくloopback(`127.0.0.1`)にバインド
   - リモートアクセス用にTailscaleまたはVPNを設定
   - ファイアウォールルールを設定
   - 強力な認証トークンを有効化

3. **ツールサンドボックス化**(Dockerが必要)
   ```json
   {
     "sandbox": {
       "mode": "all",
       "scope": "agent",
       "workspaceAccess": "ro"
     }
   }
   ```

4. **セキュリティ監査を実行**
   ```bash
   openclaw security audit --deep --fix
   ```

**よくあるセキュリティミス:**
- `dmPolicy: "open"`の使用(誰でもボットを制御可能)
- 認証なしで`0.0.0.0`でゲートウェイを公開
- ファイル権限なしでAPIキーを平文保存
- ツールサンドボックス化を有効にしない
- セキュリティ監査をスキップ

#### フェーズ4:サービス設定&監視(30〜45分)

```bash
# systemdサービスを作成
cat > /etc/systemd/system/openclaw.service <<EOF
[Unit]
Description=OpenClaw Gateway
After=network.target

[Service]
Type=simple
User=openclaw
WorkingDirectory=/opt/openclaw
EnvironmentFile=/opt/openclaw/.env
ExecStart=/usr/bin/openclaw gateway --port 18789
Restart=on-failure

[Install]
WantedBy=multi-user.target
EOF

# サービスを有効化して開始
systemctl enable openclaw
systemctl start openclaw

# ログを監視
journalctl -u openclaw -f
```

**継続的なメンテナンスが必要:**
- エラーのログ監視
- クラッシュと再起動の処理
- 依存関係の更新
- セキュリティ脆弱性のパッチ適用
- ディスク容量の管理

### 合計時間投資:2〜4時間
(すべてがスムーズに進んだ場合!)
