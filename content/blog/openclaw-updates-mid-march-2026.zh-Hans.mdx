OpenClaw 在三月的上半个月推进得非常快。从 3 月初的 [`2026.3.8`](https://github.com/openclaw/openclaw/releases/tag/v2026.3.8) 一路发布到中旬的 [`2026.3.13`](https://github.com/openclaw/openclaw/releases/tag/v2026.3.13-1)。

相比零碎的版本日志，这里为你整理了从 `3.8` 到 `3.13` 之间，普通用户最能直接感受到的几个核心变化。

---

## 终于吃满了 GPT-5.4 的百万上下文，Perplexity 搜索也回归了

随着新模型发布，`2026.3.8` 版本在底层做了一次关键的兼容性调整：
- **完整支持 GPT-5.4**：不再沿用旧版 Codex 的长度限制，彻底放开了高达 1,050,000 token 的上下文窗口，处理超长文档或巨型代码库时不再会无故截断。
- **Perplexity 搜索回归**：全面恢复了对 Perplexity OpenRouter / Sonar 模型的支持。无论你是填原生 API Key 还是走代理，现在都能直接让 Agent 连上 Perplexity 进行高质网页检索了。

---

## 聊天界面顺了不少，Mac 语音也更稳了

这半个月客户端体验有几个明显的改善：

- 工具执行很多的时候，聊天记录不会频繁整页重载了。
- 很长的回复不会再挤成一块难读的大灰色区域了。
- `2026.3.12` 重构了 dashboard 的布局，聊天、配置、会话管理更集中。
- Mac 用户的福音：`2026.3.8` 修复了 macOS 上 VoiceWake 和 Talk 语音覆盖层的闪退问题，直接动嘴聊天的体验更好。

这类改动不算炫技，但能解决“界面一顿一顿、消息看着费劲”的痛点。

---

## Slack 终于能发按钮了，Teams 权限更严密

如果你把 OpenClaw 接到了办公软件，这段时间有几个好消息：

- **Slack**：`2026.3.12` 和 `3.13` 终于加入了 Slack Block Kit 的支持。机器人的回复不再只是一坨纯文字，而是可以带有按钮、卡片、下拉菜单。在团队里用来做确认和审批，它终于像个真正的工作助手了。
- **MS Teams**：`2026.3.8` 修复了白名单机制，现在即使配置了频道和群组级许可，也不会意外将群里的陌生人放行，安全性大幅提高。

---

## Telegram 和 Discord 补上了一大批坑

渠道稳定性经过几个大版本的修补，现在更扛造了：

- **Telegram**：收图和文件的下载链路继续补强；改进了重连时的清理逻辑，不再会出现“机器人重启后，重复收到旧消息”的问题。
- **Discord**：回复拆分、状态反馈、路由稳定性都在持续修。
- `/reset` 之后，原来的账号和线程信息会尽量保住，不容易“聊着聊着串线”。

如果你之前遇到过图片处理失败、长回复切得乱、群聊重复发消息等问题，这一波大体都在往这些地方打补丁。

---

## 定时任务更值得重新试一次了

很多人用 OpenClaw 是为了让它每天自动跑早报、搞汇总。从 `3.8` 开始，定时任务（Cron）连续做了几轮大修：

- 修复了因为服务重启导致的漏发问题（现在恢复后会合理补发，不再一窝蜂挤爆或直接跳过）。
- `2026.3.11` 处理了重试、状态记录边界。
- 修复了嵌套 cron 导致的系统死锁。

简单说：如果你之前觉得定时任务“偶尔会抽风”，这段时间的更新就是在集中修这个。有些以前不太敢托付给它的日常工作，可以重新试一遍了。

---

## 手机端用起来更像个正式应用了

移动端在最近的版本里也有明显迭代：

- iOS 补了更完整的首开引导和首页布局，工具栏更适配小屏。
- Android 重做了聊天设置页，连线、语音、媒体这些选项整理得更清楚。
- onboarding（初始化配对）时的扫码和连线流程也更顺畅。

这决定了你愿不愿意把它当成每天都会打开几次的手机 App，而不是离了电脑就没法用的东西。

---

## 自托管用户的小彩蛋

如果你是自己部署的，除了 Docker 现在支持 `OPENCLAW_TZ` 时区设置以外，`3.8` 版本还自动检测 SELinux 权限并修复了 Podman 挂载失败（`Permission denied`）的问题。折腾环境的门槛又降了一点。

---

## ClawSimple 用户不需要做什么

新版本我们会在合适的时机帮你自动更新。如果你想马上用上最新的大窗口模型和这些交互优化，到 Profile 页面 Relaunch 一下就好了。
