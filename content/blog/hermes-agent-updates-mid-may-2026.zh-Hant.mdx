過去一個月 Hermes Agent 的開發節奏堪稱驚人。四個版本——[v0.11](https://github.com/NousResearch/hermes-agent/releases/tag/v2026.4.23)（Interface）、[v0.12](https://github.com/NousResearch/hermes-agent/releases/tag/v2026.4.30)（Curator）、[v0.13](https://github.com/NousResearch/hermes-agent/releases/tag/v2026.5.7)（Tenacity）和 [v0.14](https://github.com/NousResearch/hermes-agent/releases/tag/v2026.5.16)（Foundation）——在四月底到五月中旬之間落地，合計超過 3,900 次提交、2,500 個合併 PR，關閉了 500 多個 Issue。

以下是你真正能感受到的功能。

---

## 多 Agent Kanban：把任務交給 AI 團隊

v0.13 釋出了最受期待的功能之一：持久化的多 Agent Kanban 看板。

- 建立一個看板，把任務放上去，讓多個 Hermes Worker 領取、轉交和完成任務
- 心跳、殭屍檢測和重試預算保持團隊誠實——如果一個 Worker 靜默了，另一個會認領任務
- 幻覺門禁和異常退出自動阻斷，防止垃圾進入你的「已完成」列
- 一次安裝，多個 kanban——為不同專案執行獨立的看板

配合 `/goal`（v0.13），透過 Ralph 迴圈將 Agent 鎖定在目標上跨輪次執行，Hermes 現在中途偏離任務的可能性大大降低。

---

## 自主 Curator：你的 Agent 會自我維護

v0.12 引入了 `hermes curator`——一個後臺 Agent，在 Gateway 的 cron 定時器上執行，按自己的計劃維護你的技能庫：

- **按品質和使用情況對技能評分**
- **整合**相關技能，減少數量、提高品質
- **修剪**不再使用的廢棄技能
- **寫出執行報告**，供你稽核變化
- 縱深防禦門禁保護內建和 Hub 技能免受篡改
- `hermes curator status` 按使用量排序技能——從最常用到最少用

自我改進迴圈（Hermes 在每次對話後決定記住什麼的核心機制）也在 v0.12 中得到了重大升級：基於評分標準的評估取代了自由形式的判斷，派生程式優先選擇 Agent 剛載入的技能，並且即時執行時憑證能正確傳播，使迴圈擁有與 Agent 相同的工具訪問許可權。

## xAI Grok 透過 SuperGrok OAuth 接入——1M 上下文視窗

v0.14 將 Grok 作為一等公民提供商引入 Hermes。如果你付費訂閱了 SuperGrok，現在就可以透過登入 xAI 帳戶在 Hermes 中使用 Grok——無需 API Key，無需單獨付費。

- **grok-4.3 獲得 1M token 上下文視窗**——將整個程式碼庫或研究語料放進單個提示詞中
- OAuth 流程優雅處理授權錯誤
- 提供 SSH 隧道檔案，幫助在遠端機器上完成 OAuth 流程

---

## OpenAI 相容的本地代理：一份訂閱，所有工具

同樣在 v0.14 中：`hermes proxy` 啟動一個本地 HTTP 端點，對外提供 OpenAI API 格式，但後端由你登入的任意 OAuth 提供商驅動——Claude Pro、ChatGPT Pro、SuperGrok。

實際意義：Codex CLI、Aider、Cline、Continue，以及任何期望 OpenAI 相容端點的自定義指令碼，現在都可以用你現有的訂閱直接工作。無需任何 API Key。

## 瘦身浪潮：更輕、更快的安裝

v0.14 讓 Hermes 大幅變輕：

- **重型後端現在首次使用時才延遲安裝**——訊息介面卡 SDK、影像生成 SDK 和 TTS 提供商不再在安裝時全部拉取
- **`pip install hermes-agent` 可從 PyPI 安裝**——乾淨的安裝不再拖入每個可選依賴
- **分層安裝**在你的平臺拒絕某個 wheel 時優雅降級
- **冷啟動縮短約 19 秒**——疊加 v0.12 中 TUI 冷啟動約 57% 的縮減
- **瀏覽器 CDP 呼叫快了 180 倍**——自動化操作感覺即時而非卡頓
- **跨會話 1 小時 Claude 提示詞快取**——重複推理更便宜也更快
